# Query Fan-Out Generator — Maximus

The **Query Fan-Out Generator** is an elite technical utility designed to show how AI search engines (like ChatGPT Search, Google AI Mode, and Perplexity) decompose complex user prompts into multiple sub-queries (fan-out) to fetch accurate web data.

**Live URL:** [maximuslabs.ai/tools/query-fan-out-generator](https://www.maximuslabs.ai/tools/query-fan-out-generator)

Built for SEOs, content strategists, and keyword researchers looking to understand the "hidden" search patterns of generative engines.

---

## 🚀 Quick Start

### 1. Prerequisites
- Node.js (v18+)
- [Groq API Key](https://console.groq.com/keys) (Free)
- *Optional:* [SerpApi Key](https://serpapi.com) (For live search results)

### 2. Setup
Clone the repository and install dependencies:
```bash
npm install
```

### 3. Environment Variables
Create a `.env` file in the root directory:
```env
# Required for query decomposition
GROQ_API_KEY=your_groq_key_here

# Optional: Required for live search result previews
SERPAPI_API_KEY=your_serpapi_key_here
```

### 4. Running the App
The application requires two processes to run:

**Start the Backend API:**
```bash
npm run server
```

**Start the Frontend (Vite):**
```bash
npm run dev
```

The app will be available at [http://localhost:5173](http://localhost:5173).

## ✨ Features

- **Multi-Engine Simulation**: Choose between ChatGPT Search, Google AI Mode, and Perplexity logic.
- **Hidden Query Extraction**: See the exact sub-queries generated by the LLM.
- **Live Search Previews**: If SerpApi is configured, view live Google results directly inside the app.
- **Strategic Reasoning**: Get a brief explanation of why specific sub-queries were generated for each engine.
- **One-Click Copy**: Easily export all generated queries for your own research.

## 🛡️ Deployment & Production Readiness

This tool is built to be production-ready with the following security and stability features:

- **Rate Limiting**: Protection against API abuse with specific tiers for LLM and Search requests.
- **Security Headers**: Powered by `helmet` to manage CSP, HSTS, and other security headers.
- **Payload Validation**: Input length and payload size limits on all endpoints.
- **Production Caching**: In-memory caching for both LLM responses and search results to reduce latency and costs.
- **Single-Package Serving**: The backend serves the built frontend (`dist/`) directly, simplifying deployment to platforms like Heroku, Render, or Railway.
- **CORS Management**: Configurable allowed origins via environment variables.

### Environment Variables

For production, set the following in your environment:

| Variable | Description | Recommendation |
| :--- | :--- | :--- |
| `GROQ_API_KEY` | Your Groq API key | **Required** |
| `SERPAPI_API_KEY` | Your SerpApi key | Optional (for live search) |
| `PORT` | Server listening port | Default: `3001` |
| `ALLOWED_ORIGINS` | Comma-separated list of allowed CORS origins | Default: `http://localhost:5173,http://localhost:3000` |
| `NODE_ENV` | Environment identifier | Set to `production` |

### Deployment Steps

1. **Build the Frontend**:
   ```bash
   npm run build
   ```
2. **Start the Server**:
   ```bash
   NODE_ENV=production npm run server
   ```
